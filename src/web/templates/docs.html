Overview

The Image Metadata Viewer API extracts metadata from remote image URLs and uploads. Use GET for quick single lookups or POST for batch requests and file uploads.

GET /api/{url} POST /api

Base URL

{{.BaseURL}}/api

Max image size: 20 MB. Timeout: 15s. Supported formats: JPG, PNG, GIF, BMP, TIFF, WebP.

Endpoints

GET /api/{url}

Fetch metadata for a single remote image URL.

curl "{{.BaseURL}}/api/https://example.com/image.jpg"

Tip: Use POST if the URL contains query or hash fragments.

POST /api

Send JSON with multiple URLs or upload files.

curl -X POST {{.BaseURL}}/api \
  -H "Content-Type: application/json" \
  -d '{"urls":["https://example.com/a.jpg","https://example.com/b.png"]}'
curl -X POST {{.BaseURL}}/api \
  -F "files=@image1.jpg" \
  -F "files=@image2.png"

Sample Response

{
  "success": true,
  "data": [
    {
      "fileName": "image.jpg",
      "fileSize": 164000,
      "fileSizeHuman": "160.2 KiB",
      "fileType": "JPEG",
      "fileTypeExtension": "jpg",
      "mimeType": "image/jpeg",
      "width": 1920,
      "height": 1080,
      "aspectRatio": "1.778",
      "aspectRatioFraction": "16:9",
      "megapixels": 2.0736,
      "source": "remote",
      "status": "200 OK",
      "duration": "156ms"
    }
  ]
}

Error Response

{
  "success": false,
  "error": "Invalid URL format"
}

Fields

  • Dimensions: width, height, aspectRatio, aspectRatioFraction
  • File: fileName, fileSize, fileType, mimeType
  • Remote: status, finalURL, duration, lastModified
